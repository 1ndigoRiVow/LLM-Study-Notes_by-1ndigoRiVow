# 梯度下降 (Gradient Descent)

假设你深夜在山顶，想下山回家。四周漆黑一片，你只能靠脚感知地面的坡度。梯度下降就是这样一个过程：每次都朝最陡的下坡方向迈一步，最终到达山底（最低点）。

在机器学习中，这个“山”就是损失函数（衡量模型预测与真实值的差距），“山底”就是最优参数（使损失最小的参数）。

---

## 1. 核心原理

泰勒展开的线性近似梯度下降的本质是：在当前点寻找一个局部方向，使得函数值下降最快。

从数学分析来看，对于可微函数 $f(x)$，其在 $x_0$ 处的一阶泰勒展开为：
$$f(x) \approx f(x_0) + f'(x_0)(x - x_0)$$

我们要寻找一个位移 $\Delta x = x - x_0$，使得 $f(x) < f(x_0)$。

令 $\Delta x = -\eta f'(x_0)$（其中 $\eta > 0$ 是步长），带入上式：
$$f(x_0 - \eta f'(x_0)) \approx f(x_0) - \eta [f'(x_0)]^2$$

由于 $[f'(x_0)]^2 \ge 0$，只要步长 $\eta$ 足够小，函数值一定会在 $x_0$ 处下降。这就是梯度下降的数学合法性来源。

## 2. 定义与算法流程定义

梯度下降是一种迭代优化算法，通过不断沿着目标函数 $J(\theta)$ 在当前点的负梯度方向更新参数 $\theta$，以求得函数的局部最小值。

### 通用算法步骤：

1. 初始化：随机选择初始参数 $\theta_0$，设定学习率 $\eta$ 和收敛精度 $\epsilon$。

2. 计算梯度：计算当前位置的梯度 $g_t = \nabla J(\theta_t)$。

3. 更新参数：$\theta_{t+1} = \theta_t - \eta \cdot g_t$。

4. 循环：重复步骤 2-3，直到 $|\nabla J(\theta)| < \epsilon$ 或达到最大迭代次数。

## 3. 从一元到多元的演进

### A. 一元梯度下降 (Scalar Case)

设 $J(w) = w^2$，求最小值。

梯度（导数）：$J'(w) = 2w$

更新公式：$w_{t+1} = w_t - \eta(2w_t)$

### B. 二元梯度下降 (2D Case)

设 $J(w_1, w_2) = w_1^2 + w_2^2$。此时梯度是一个向量：
$$\nabla J = \begin{bmatrix} \frac{\partial J}{\partial w_1} \\ \frac{\partial J}{\partial w_2} \end{bmatrix} = \begin{bmatrix} 2w_1 \\ 2w_2 \end{bmatrix}$$

更新公式：$\begin{bmatrix} w_1 \\ w_2 \end{bmatrix}_{new} = \begin{bmatrix} w_1 \\ w_2 \end{bmatrix}_{old} - \eta \begin{bmatrix} 2w_1 \\ 2w_2 \end{bmatrix}$

几何直觉：在等高线图中，梯度向量始终垂直于等高线并指向高处，而负梯度则指向“坑底”。

### C. 多元梯度下降 (General Case / Matrix Form)

在大模型中，$\theta$ 是拥有数十亿参数的向量。以MSE损失函数为例：

$$J(\theta) = \frac{1}{2m} \sum_{i=1}^m (h_\theta(x^{(i)}) - y^{(i)})^2$$

矩阵化表示：$\nabla_\theta J(\theta) = \frac{1}{m} X^T(X\theta - y)$

注：这与计量经济学中 OLS 的正规方程 $X^T X \hat{\beta} = X^T y$ 的推导逻辑完全一致。

## 4. Python 代码实现 (NumPy 版)

以简单的线性回归 $y = wx + b$ 为例：

```python
import numpy as np

def gradient_descent(X, y, lr=0.01, epochs=100):
    m, n = X.shape  # m个样本, n个特征
    w = np.zeros(n) # 初始化权重
    b = 0           # 初始化截距
    
    for _ in range(epochs):
        # 1. 预测
        y_pred = np.dot(X, w) + b
        
        # 2. 计算梯度 (均方误差 MSE 的导数)
        dw = (1/m) * np.dot(X.T, (y_pred - y))
        db = (1/m) * np.sum(y_pred - y)
        
        # 3. 更新参数
        w -= lr * dw
        b -= lr * db
        
    return w, b
```

## 5. 梯度下降的变体 (在大模型中的进化)

为了解决“速度慢”和“局部陷阱”问题，社区开发了三大变体：

**BGD (Batch)：**使用全部数据算一次梯度。最准，但大数据下内存会爆。

**SGD (Stochastic)：**每次只用一个样本更新。极快，但路径由于噪声极其震荡。

**MB-GD (Mini-batch)：**每次用一小批数据（如 128 个）。这是深度学习的标准做法，平衡了计算效率和梯度稳定性。需要额外调节Batch Size。

## 6. 应用场景

**神经网络训练：**大模型（LLM）的所有参数更新，底层全是各种变体梯度下降。

**经济学计量模型：**当数据量极其巨大，无法直接求矩阵逆（$X^T X$ 逆不动）时，我们会改用梯度下降求 OLS 的数值解。

**逻辑回归 (Logistic Regression)：**在金融信贷风险模型中，梯度下降是求解最大似然估计（MLE）的标准方法。

## 7. 给经济学背景的深度思考：为什么不直接求导为0？

在计量经济学中，通常习惯令 $J'(\theta) = 0$ 直接求解析解。为什么大模型非要用梯度下降迭代？

**非凸性：**神经网络的 Loss 表面是波浪形的，求导为 0 的点可能是极大值、极小值或鞍点。

**计算成本：**解析解通常涉及矩阵求逆，时间复杂度为 $O(n^3)$。对于 175B 参数的 GPT，矩阵求逆是不可能完成的任务，而梯度下降的一阶更新只需 $O(n)$。
